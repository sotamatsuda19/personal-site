Sota's Portfolio Site
=====================

A static personal portfolio website built with React 18 and Tailwind CSS.
JSX is pre-compiled with esbuild and CSS is generated by the Tailwind CLI,
so the browser loads only lightweight, production-ready files.
Supports English and Japanese (日本語) with a language toggle in the navbar.


Prerequisites
-------------

Node.js (v18+) and npm are required for the build step.

Install dependencies once:

  npm install


Building
--------

One-time production build:

  npm run build

Watch mode (rebuilds CSS and JS on every save):

  npm run dev

Both commands compile src/app.jsx -> dist/app.js (via esbuild) and
src/styles.css -> dist/styles.css (via Tailwind CLI, minified).


Running Locally
---------------

1. Build the project:

   npm run build

2. Start a local server (Python 3, npx serve, VS Code Live Server, etc.):

   python3 -m http.server 3001

3. Open http://localhost:3001 in your browser.


Page Overview
-------------

Hero:
  Full-screen landing section with the site title and a button that links
  to the Works section. When artwork images are available, they cycle as a
  background slideshow with a smooth crossfade every 4 seconds. A
  semi-transparent white overlay keeps text readable. When no artwork
  images are set, the background is plain white.

Works:
  A unified grid combining both projects and artwork into a single section.
  Cards are only visible when they have an image (src) set. If no cards
  have images, the section shows "Coming soon..." / "準備中..." instead.
  Each card displays an image, title, and a one-line description.

Profile:
  Short bio section with links to GitHub and note.

Contact:
  Contact section with an email link.

Navbar:
  Fixed top bar with links to Profile, Works, and Contact. On desktop,
  links are shown inline. On mobile (< 640px), a hamburger menu toggles
  a dropdown. A language toggle button (日本語 / EN) is shown on both
  desktop and mobile. Toggling the language also updates the <html lang>
  attribute for accessibility.


Deploying
---------

Run npm run build first, then deploy the root folder.

GitHub Pages:
  1. Create a new GitHub repo (e.g. "portfolio" or "username.github.io")
  2. Push all files in this folder to the repo
  3. Go to Settings > Pages > set source to the main branch
  4. Your site will be live at https://username.github.io/portfolio/

Netlify:
  1. Go to https://app.netlify.com
  2. Drag and drop this entire folder onto the deploy area
  3. Done — Netlify gives you a live URL immediately


Project Structure
-----------------

portfolio-site/
  index.html              Main HTML file. Loads React via CDN, and
                          references the built CSS and JS from dist/.

  src/
    app.jsx               The React app source (JSX). Contains:
                          - i18n system (STRINGS dictionary, LangContext)
                          - Data arrays at the top (PROJECTS, ARTWORK)
                            with bilingual title/description fields
                          - Navbar with language toggle and mobile menu
                          - Hero with artwork slideshow, unified Works
                            grid, profile, and contact sections

    styles.css            Tailwind directives and custom CSS (animations,
                          hero slideshow crossfade, scrollbar styles).

  dist/                   Build output (generated by npm run build):
    app.js                Pre-compiled JS (no Babel needed at runtime)
    styles.css            Production Tailwind CSS (only used classes)

  assets/
    screenshots/          Drop project screenshots here, then reference
                          them in the PROJECTS array in src/app.jsx
    artwork/              Drop artwork images here, then add entries to
                          the ARTWORK array in src/app.jsx

  tailwind.config.js      Tailwind configuration (fonts, shadows)
  package.json            Build scripts and dev dependencies
  .gitignore              Ignores node_modules/ and .DS_Store
  README.txt              This file


Adding Content
--------------

All content is managed through arrays at the top of src/app.jsx. Each
entry uses bilingual title and description fields.

Add a project:
  Drop the image in assets/screenshots/, then add to the PROJECTS array:
  {
    title: { en: "My Project", ja: "私のプロジェクト" },
    src: "assets/screenshots/my-project.png",
    description: { en: "What it does.", ja: "これの説明。" },
  }

Add artwork:
  Drop the image in assets/artwork/, then add to the ARTWORK array:
  {
    title: { en: "Piece Title", ja: "作品タイトル" },
    src: "assets/artwork/piece.jpg",
    description: { en: "About this piece.", ja: "この作品について。" },
  }

After editing src/app.jsx, run npm run build to regenerate dist/app.js.

Cards without a src set are hidden. The Works section shows "Coming soon..."
only when no cards have images at all.

Images with a valid src in the ARTWORK array will also appear in the hero
background slideshow.


i18n (Internationalization)
---------------------------

All UI strings are stored in the STRINGS object at the top of src/app.jsx,
keyed by language code ("en" and "ja"). To update UI text (headings,
buttons, bio, etc.), edit the corresponding entry in STRINGS.

The language toggle button in the navbar switches between English and
Japanese. The current language is shared across all components via
React Context. The <html lang> attribute is kept in sync automatically.
